## 2. 当前需求理解

项目目标是做一个校园碳排放/低碳行为分析系统，重点不是单纯后台管理，而是把学生行为、位置、建筑水电数据、预测模型结果整合到一个可视化页面里。

核心业务链路：

1. 上传或接入校园图片/视频。
2. 根据图片识别学生行为和位置，例如是否关灯、用水、插座/充电器、教室/宿舍/楼栋场景。
3. 将识别结果与楼栋、专业类别、低碳行为问卷得分关联。
4. 结合 MySQL 中的建筑水电数据、预测结果和行为统计。
5. 在前端展示：
   - 左侧：图片上传、识别结果、学生行为列表。
   - 中间：关系图/示意图，展示“学生行为 - 位置 - 楼栋 - 水电/碳排”的联系。
   - 右侧：按学校楼栋列出的用电、用水、预测值、误差数据表。
   - 最右侧或下方：用水用电统计图、建筑趋势图、行为得分图。

## 3. 推荐技术栈

### 后端

| 模块 | 技术 | 原因 |
| --- | --- | --- |
| Web API | FastAPI | Python 原生，接口清晰，适合文件上传、模型推理和数据接口。 |
| ASGI 服务 | Uvicorn | FastAPI 标准运行方式。 |
| 数据库 | MySQL 8.0 本地连接 | 符合要求，本机已检测到 MySQL80 服务运行。 |
| ORM | SQLAlchemy 2.x | 可读性高，后续迁移和查询维护方便。 |
| 迁移 | Alembic | 管理表结构版本。 |
| 配置 | `.env` + pydantic-settings | 避免把 MySQL 密码写死在代码中。 |

### 图像识别/算法

| 模块 | 技术 | 说明 |
| --- | --- | --- |
| 图像处理 | OpenCV + Pillow | 图片读取、裁剪、格式转换、标注框绘制。 |
| 行为识别模型 | Ultralytics YOLO | 先用于人、物体、场景目标检测；后续可用自有校园数据微调。 |
| 深度学习 | PyTorch | 已有原型脚本使用 PyTorch/GAT，继续沿用。 |
| 预测模型 | PyTorch GAT/时间序列模型 | 原型脚本已经实现建筑图关系与水电预测，可重构为服务模块。 |
| 传统数据处理 | pandas、numpy、scikit-learn | CSV/Excel 读取、归一化、指标计算。 |

### 前端

| 模块 | 技术 | 说明 |
| --- | --- | --- |
| 应用框架 | Vue 3 + Vite + TypeScript | 适合快速做仪表盘，结构清晰。 |
| 图表 | Apache ECharts | 折线图、柱状图、关系图、表格联动。 |
| 地图/平面图可视化 | ECharts Graph 或 SVG/Canvas | 先按草图做关系图，后续可替换为真实校园平面图。 |
| 数据表 | Element Plus Table | 楼栋水电数据表、排序、筛选。 |
| HTTP | Axios | 连接 FastAPI 接口。 |

## 7. 初始开发顺序

1. 建立后端 FastAPI 项目结构、`.env.example`、MySQL 连接检查接口。
2. 导入 `prediction_results.csv` 和 `behavior_summary(1).csv` 到 MySQL。
3. 把原型脚本拆成可读模块：数据预处理、模型、训练、预测、指标输出。
4. 建立图片上传和 YOLO 识别接口。
5. 建立 Vue 仪表盘：左侧识别结果，中间关系图，右侧楼栋水电表，统计图。
6. 等真实 Excel、楼栋平面图、行为训练数据补齐后，再训练/微调识别模型。
