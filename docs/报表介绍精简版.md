# 报表介绍精简版

用于答辩时口头说明 `docs/ppt-html/index.html` 的报表逻辑，保留当前版本的核心口径，但改成更短、更好讲的表达。

## 1. 顶部指标卡片

- 数据来源：`energy_predictions + recognition_samples + behavior_impact_rules + buildings`
- 计算链路：`最新批次汇总 -> 历史均值对比 -> 趋势指数`
- 直接解释：
  - `识别样例数 = recognition_samples.count(*)`
  - `规则库数量 = behavior_impact_rules.count(*)`
  - `覆盖楼栋数 = buildings.count(*)`
- 答辩一句话：
  - 这组卡片先说明系统有没有数据、覆盖多少楼、当前能不能演示识别。

## 2. 楼栋预测概览表

- 数据来源：`buildings + energy_predictions`
- 计算链路：
  - `最新 time_step -> 当前批次`
  - `用电水平 = electricityActual / maxElectricity * 100`
  - `用水水平 = waterActual / maxWater * 100`
  - `排序 = 用电↓ -> 用水↓ -> 综合分`
- 偏差展示：
  - `deviationRaw = (electricityError + waterError) / (electricityActual + waterActual) * 100`
  - `deviationDisplay = deviationRaw * 0.45`
- 答辩一句话：
  - 这张表不展示敏感原值，只展示相对水平和预测稳定性，方便快速指出重点楼栋。

## 3. 楼栋气泡图

- 数据来源：`buildings + energy_predictions`
- 计算链路：
  - `楼栋坐标 -> 气泡位置`
  - `实际用电占比 -> 气泡大小`
  - `强度分级 -> 气泡颜色`
- 答辩一句话：
  - 这张图把“哪栋楼能耗高”放回校园空间里看，便于定位重点区域。

## 4. 趋势图

- 数据来源：`energy_predictions`
- 计算链路：
  - `同一 time_step 的全部楼栋实际值求和 -> 全校实际趋势`
  - `同一 time_step 的全部楼栋预测值求和 -> 全校预测趋势`
- 答辩一句话：
  - 趋势图看的是全校整体变化，不是单栋楼，主要用于说明模型预测和真实变化是否同步。

## 5. 行为影响占比图

- 数据来源：`behavior_impact_rules`
- 计算链路：
  - `行为权重 = |electricityFactor| + |waterFactor|`
  - `行为占比 = 单行为权重 / 全部行为权重总和`
- 答辩一句话：
  - 这张图回答的是“哪些低碳行为最值得优先倡导”。

## 6. 上传识别结果

- 数据来源：`recognition_samples` 或真实上传后的识别结果表
- 计算链路：`上传图片 -> 识别行为 -> 返回置信度/位置/影响描述`
- 答辩一句话：
  - 这一块体现系统不是静态看板，而是能形成“上传 -> 识别 -> 分析”的闭环。
