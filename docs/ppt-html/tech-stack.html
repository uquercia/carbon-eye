校园低碳行为识别与水电预测平台技术栈说明

本系统围绕“校园图片识别、楼栋水电预测、低碳行为分析、数据可视化展示”四个核心环节搭建。整体采用前后端分离架构，前端负责交互和图表展示，后端负责数据聚合、文件上传、识别任务调度和接口输出，数据库统一保存楼栋、预测、行为和识别结果数据。

WEB
Vue 3 + TypeScript构建驾驶舱页面、图表交互和上传识别入口。
API
FastAPI + SQLAlchemy提供数据聚合接口、上传接口和异步识别任务接口。
DB
MySQL 8保存楼栋、预测、行为得分、行为规则、图片和识别结果。
OPS
Docker + Nginx将前端、后端、数据库容器化部署，并通过统一域名对外访问。

前后端技术栈

项目采用前后端分离模式。浏览器只负责展示和交互，后端统一管理数据库查询、上传文件、识别任务和数据聚合，保证页面展示口径和数据库口径一致。

frontend

前端展示层

  • 使用 Vue 3 组织页面组件，提升驾驶舱各模块的复用性。
  • 使用 TypeScript 约束接口数据结构，减少字段传递错误。
  • 使用 Vite 构建前端资源，保证开发和部署构建速度。
  • 使用 ECharts 展示趋势图、柱状图、饼图和楼栋能耗关系图。
  • 使用 Element Plus 完成表格、进度条、标签等基础交互组件。
backend

后端服务层

  • 使用 FastAPI 提供 REST 接口，包含驾驶舱数据、楼栋数据、趋势数据和上传识别接口。
  • 使用 SQLAlchemy 作为 ORM 层，将数据库表映射为 Python 模型，便于维护查询逻辑。
  • 使用 Pydantic 定义接口输出结构，使前后端字段对应更清晰。
  • 上传校园图片后创建识别任务，前端通过轮询接口获取识别进度和结果。
deploy

部署与访问层

  • 使用 Docker Compose 管理 MySQL、API、Web 三个服务容器。
  • 使用 Nginx 承载前端静态资源，并把 API 请求反向代理到后端容器。
  • 上传文件通过后端保存到服务器目录，再由统一访问路径提供给前端展示。
  • 答辩 HTML 页面作为静态页面随 Web 镜像一起发布，便于外部访问。

技术栈流程图

系统从原始数据和校园图片进入后端，经数据库、模型和聚合接口处理后，最终在前端驾驶舱中形成可解释的图表结果。

数据接入导入楼栋水电预测结果、低碳行为问卷、训练结果图片，并接收用户上传的校园图片。
数据库存储MySQL 保存楼栋、预测、行为得分、行为规则、上传图片、识别任务和识别结果。
模型分析水电预测模型输出实际/预测对比；自训练行为识别模型分析校园图片中的低碳行为和影响。
后端聚合FastAPI 按最新批次、时间趋势、行为规则和识别结果聚合为统一接口。
前端展示Vue 驾驶舱将数据渲染为指标卡、楼栋气泡图、趋势图、行为占比图和识别结果卡片。

数据库表结构关系图

数据库以楼栋和识别任务为两条主线：一条支撑水电预测和空间展示，另一条支撑图片上传、识别分析和低碳行为解释。

buildings楼栋基础信息：楼栋名称、区域、专业映射、地图坐标。为能耗预测和楼栋气泡图提供基础维度。
energy_predictions水电预测结果：时间批次、楼栋编号、实际用电/用水、预测用电/用水、误差值。
behavior_scores专业行为得分：各专业类别在多项低碳行为问卷中的得分和综合得分。
training_images训练结果图片：保存模型训练曲线、预测对比图和行为得分图的访问地址。
behavior_impacts行为影响规则：每类低碳行为的说明、用电影响因子和用水影响因子，用于行为占比图。
recognition_samples识别样例数据：行为名称、位置、楼栋、置信度、影响摘要和水电影响估计。
uploaded_images上传图片记录：原始文件名、服务器文件名、访问地址、文件类型和上传时间。
recognition_tasks识别任务记录：关联上传图片，保存任务状态、模型版本、错误信息和完成时间。
recognition_results识别结果记录：关联识别任务，保存行为名称、位置、置信度、影响摘要和水电变化估计。
关系 1buildings 与 energy_predictions、recognition_samples 通过楼栋编号关联。
关系 2uploaded_images 与 recognition_tasks 通过图片编号关联，任务再关联多条 recognition_results。
关系 3behavior_impacts 和 behavior_scores 为行为分析图表提供解释规则和人群得分。

项目亮点

项目不是单纯展示静态图表，而是把数据预测、行为识别、数据脱敏展示和云端部署组合成一套完整的校园低碳分析链路。

亮点一：水电预测结果可视化系统将楼栋水电实际值、预测值和误差统一入库，并按时间批次聚合成全校趋势图，能够展示预测结果与真实变化之间的拟合情况。
亮点二：自训练行为识别模型接入校园图片上传后进入识别任务队列，由自训练低碳行为识别模型分析图片中的行为、位置和置信度，再转化为对用电、用水影响的文字说明和结构化结果。
亮点三：敏感水电数据脱敏展示页面不直接暴露原始水电数值，而是通过趋势指数、进度条、气泡大小和偏差等级表达相对强弱，既能支持答辩展示，也降低原始数据泄露风险。
亮点四：前后端接口统一聚合前端首页只需要请求驾驶舱聚合接口，就能拿到楼栋、趋势、行为得分、行为规则、识别样例和训练图片，降低页面与数据库之间的耦合。

从“数据展示”升级为“行为分析”

系统把楼栋水电预测、低碳行为问卷、自训练行为识别模型和校园图片上传连接在一起，形成“上传图片、识别行为、计算影响、图表展示”的闭环。答辩展示时，教师看到的不只是图表结果，还能看到每个图表背后的数据来源、计算逻辑和系统实现路径。

技术实现口径：前端使用 Vue 3、TypeScript、Vite、ECharts、Element Plus；后端使用 FastAPI、SQLAlchemy、Pydantic、PyMySQL；数据库使用 MySQL 8；部署使用 Docker Compose 和 Nginx；算法部分包括 水电预测模型、自训练低碳行为识别模型和行为影响规则计算。
