part 01 / 楼栋怎么统计

楼栋类报表先看最新时间步，再做分布、排序和偏差判断

数据来源

  • 这一批数据就是“当前楼栋状态”，供地图、排行榜和表格共用。

这一页讲楼栋类报表。它的关键不是把每栋楼都长期展开，而是先抓当前最新一批状态。后端先找最大时间步，再把这个时间步下所有楼栋的电、水实际值和预测值一起拿出来。前端在这批数据上做三件事：第一是地图气泡，按坐标摆位置，按实际用电决定气泡大小；第二是高用电排行，按实际用电从高到低取前五；第三是表格概览，把电水占比做归一化，再结合误差形成一个展示偏差等级。所以老师看到的不只是“哪栋楼高”，还能看到“高到什么程度”和“预测稳不稳”。

part 02 / 趋势和行为怎么统计

趋势图看校园整体变化，行为图看人群差异和干预优先级

趋势图

  • 按 time_step 分组。
  • 把所有楼栋在同一时间步的实际值、预测值分别求和。
  • 前端再做 0~100 归一化，强调“走势是否一致”。

专业得分图

  • 问卷汇总后，每个专业类别形成一个综合得分。
  • 按专业编码顺序展示，不额外改排名。
  • 用于比较不同群体的低碳行为表现差异。

行为影响饼图

  • 每个行为都有用电因子和用水因子。
  • 前端按 |电因子| + |水因子| 重算权重。
  • 权重越大，越值得优先做倡导和干预。
趋势图回答“整体消耗有没有按预期变化”
得分图回答“不同专业群体差异在哪”
影响图回答“先抓哪些行为最有效”
这一页把整体趋势和行为统计连起来讲。趋势图的逻辑很直接：以时间步为单位，把所有楼栋的实际值和预测值分别加总，形成校园整体走势。前端再把序列归一化到 0 到 100，目的不是展示绝对数，而是看实际曲线和预测曲线的同步程度。行为统计分两块，一块是不同专业类别的综合低碳得分，它反映人群差异；另一块是行为影响权重，按用电因子和用水因子的绝对值相加来算，谁的权重大，谁就更值得优先倡导。这样报表就把“能耗变化”和“行为原因”接上了。

part 03 / 怎么形成展示闭环

最后不是只停在图表上，而是把数据汇总、识别结果和训练图放到一个可解释页面里

页面形成过程

01数据库聚合

楼栋、趋势、得分、行为规则、训练图统一整理。

02前端轻量加工

做排序、归一化、偏差等级、饼图权重和展示卡片。

03上传识别闭环

图片上传后生成任务，再返回行为名称、位置、置信度和影响说明。

支撑材料

对答辩老师来说，这一页最重要的结论是：系统展示的是一条完整链路，既能解释报表怎么来，也能说明这些统计结果最后如何支持分析和展示。

建议讲法：先讲来源，再讲规则，最后讲结论
最后一页收束成闭环。前面几页已经解释了楼栋报表、趋势图和行为图怎么形成，这里要让老师看到它们最终都汇总到同一个页面里。做法上可以简单讲三步：先把数据库中的楼栋数据、趋势数据、行为得分、行为规则和训练图片统一聚合；再由前端做轻量加工，比如排序、归一化、偏差等级和权重重算；最后加上上传图片后的识别结果卡片，让系统不只会展示历史数据，也能形成“上传图片、识别行为、返回分析”的闭环。这样答辩时重点就回到实现价值，而不是陷进技术细节。
